# Security Policy

`agent-team` is pre-v1 software and changes quickly, but security reports are
welcome and taken seriously.

## Reporting a Vulnerability

Please report suspected vulnerabilities by email:

`agentteaminbox@gmail.com`

Include enough detail to reproduce or assess the issue:

- affected version, commit, or release
- operating system and runtime profile when relevant
- steps to reproduce
- expected and actual impact
- any logs or proof-of-concept details that are safe to share

Please avoid public disclosure until maintainers have had a reasonable chance
to investigate and respond.

## Scope

The most useful reports are concrete issues in the CLI, daemon, template
rendering, worktree handling, provider integrations, mailbox/channel delivery,
or docs that could cause unsafe operator behavior.

Do not send secrets, production credentials, or private third-party data unless
you have already minimized the report to the smallest necessary detail.
